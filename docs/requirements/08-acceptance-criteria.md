# 9. 受け入れ基準

## 9.1 Phase 1 完了基準
- [ ] 機密情報の保存・取得ができる
- [ ] 機密情報が暗号化されて保存される
- [ ] トークンによるアクセス制御が機能する
- [ ] プログラムから利用できる
- [ ] 基本的な CLI コマンドが動作する
- [ ] すべてのユニットテストが通過する

## 9.2 Phase 2 完了基準
- [ ] Passkey 認証が機能する
- [ ] ユーザー登録・認証フローが完成している（Claude Code と同様の UX）
- [ ] ユーザーごとに機密情報が完全に分離されている
- [ ] 監査ログが記録される
- [ ] トークン数制限が機能する
- [ ] 有効期限管理機能が動作する

## 9.3 Phase 3 完了基準
- [ ] CLI から高度な操作ができる
- [ ] 対話的入力・確認ダイアログが機能する
- [ ] 監査ログを検索できる

## 9.4 Phase 4 完了基準
- [ ] PostgreSQL で動作する
- [ ] セキュリティ監査に合格する
- [ ] ドキュメントが完備されている
- [ ] パッケージレジストリで公開されている
